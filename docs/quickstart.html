Quick start

Five minutes from npm install to a working HTTP API.

1. Set up the project

mkdir my-app && cd my-app
npm init -y
npm install annotify
npm install --save-dev typescript @types/node

Add the scripts and "type": "module" to package.json:

{
  "name": "my-app",
  "type": "module",
  "scripts": {
    "build": "tsc",
    "start": "node dist/main.js"
  }
}

2. Add tsconfig.json

{
  "compilerOptions": {
    "target": "ES2022",
    "module": "NodeNext",
    "moduleResolution": "NodeNext",
    "rootDir": "./src",
    "outDir": "./dist",
    "experimentalDecorators": true,
    "useDefineForClassFields": false,
    "strict": true,
    "esModuleInterop": true,
    "skipLibCheck": true,
    "types": ["node"]
  },
  "include": ["src/**/*"]
}

3. Write your first controller

Create src/controllers/HelloController.ts:

import {
  RestController, GetMapping, PathVariable, HttpError,
} from 'annotify';

@RestController('/hello')
export class HelloController {

  @GetMapping('/')
  root() {
    return { message: 'Hello, world!' };
  }

  @GetMapping('/:name')
  greet(@PathVariable('name') name: string) {
    return { message: `Hello, ${name}!` };
  }
}

4. Wire up main.ts

import { AppBuilder } from 'annotify';
import { fileURLToPath } from 'node:url';
import { dirname, resolve } from 'node:path';

const __dirname = dirname(fileURLToPath(import.meta.url));

const app = new AppBuilder();
app.useLogger({ enabled: true });                    // optional: access log
app.exposeRoutes();                                // optional: GET /__annotify/routes

await app.scan(resolve(__dirname, 'controllers'));
await app.listen(3000);

5. Build and run

npm run build
npm start

Expected output:

[scanner] scanning ...\dist\controllers
[server] listening on http://127.0.0.1:3000
[router] 1 controller(s), 2 route(s) registered
[introspect] GET /__annotify/routes
[logger] enabled

6. Hit it

curl http://localhost:3000/hello/
{"message":"Hello, world!"}

curl http://localhost:3000/hello/world
{"message":"Hello, world!"}

What just happened

  1. TypeScript compiled src/controllers/HelloController.ts into dist/controllers/HelloController.js. The decorators ran at class definition time, populating a WeakMap with per-method parameter metadata and attaching a route table to the class itself under a symbol key.
  2. AppBuilder.scan() walked dist/controllers, dynamically imported each .js file, and registered any export whose class had the route metadata attached.
  3. AppBuilder.listen(3000) instantiated each controller, bound the handler methods, built a trie router, and started a native http.Server.
  4. Each request walks the trie, the resolver pulls values from the matched route's path variables and the parsed query string, and the handler runs.

Add another verb

Expand HelloController:

import {
  RestController, GetMapping, PostMapping,
  PathVariable, RequestBody, HttpError,
} from 'annotify';

@RestController('/hello')
export class HelloController {
  // ... existing methods ...

  @PostMapping('/echo')
  echo(@RequestBody() body: unknown) {
    return { received: body };
  }
}
curl -X POST http://localhost:3000/hello/echo \
     -H "Content-Type: application/json" \
     -d '{"x":1,"y":"two"}'
{"received":{"x":1,"y":"two"}}

Dev workflow: nodemon + ts-node

If you want hot-reload without a separate build step, install ts-node as a dev-dep and use a nodemon.json that loads ts-node's ESM loader:

{
  "watch": ["src"],
  "ext": "ts,json",
  "exec": "node --loader ts-node/esm src/main.ts"
}

Then run:

npm install --save-dev ts-node
npx nodemon

The --loader ts-node/esm flag is required so annotify's scanner can dynamically import() your .ts controllers. Without it, the scanner will silently register zero controllers.

If you'd rather use tsx (faster than ts-node), use this nodemon.json instead:

{
  "exec": "node --import tsx/esm src/main.ts"
}
