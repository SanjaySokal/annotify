Templates

Built-in EJS-style template engine plus a tiny pluggable engine interface. Render from a handler with res.render(name, data).

Setup

import { AppBuilder } from 'annotify';

const app = new AppBuilder();
app.set('views', './views');          // template directory
app.set('view engine', 'html');       // extension + engine name

// The built-in EJS engine is pre-registered under 'html'. No need to call
// app.engine('html', ...) unless you want to swap it.

Render from a handler

import { Controller, GetMapping } from 'annotify';

@Controller('/')
export class PageController {
  @GetMapping('/')
  index(_req, res) {
    return res.render('home', {
      title: 'annotify',
      items: ['a', 'b', 'c'],
    });
  }
}

Template syntax (EJS-style)

<h1><%= title %></h1>

<%# comment — emits nothing %>

<% if (items.length > 0) { %>
  <ul>
    <% for (let i = 0; i < items.length; i++) { %>
      <li><%= items[i] %></li>
    <% } %>
  </ul>
<% } else { %>
  <p>Nothing here.</p>
<% } %>

Tag reference

TagPurpose
<% code %>Scriptlet — runs JS, emits nothing.
<%= expr %>Escaped HTML interpolation.
<%- expr %>Raw HTML interpolation (no escape).
<%# comment %>Comment, emits nothing.

Pluggable engines

Register a custom engine via app.engine(name, engine). The engine receives the template source string and a data object. 

import type { EngineFn } from 'annotify';

const handlebarsAdapter: EngineFn = {
  render(template: string, data: Record<string, unknown>): string {
    return myHandlebarsCompile(template)(data);
  },
};

app.engine('hbs', handlebarsAdapter);
app.set('view engine', 'hbs');

Helpers

  • htmlEscape(value) — escape &, <, >, ", ' for safe interpolation.
  • resolveView(name, viewsDir, ext) — resolve a view name to an absolute path, adding the configured extension.
  • renderFile(name, data, viewsDir, ext, engine) — read & compile a view, returning the rendered string.

Caching

Compiled templates are cached by absolutePath + mtimeMs (Vite-style invalidation). No manual cache eviction needed.
