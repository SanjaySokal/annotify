Sample controllers

Two complete controllers from the annotify example project, with annotation-by-annotation commentary.

UserController

examples/user.controller.ts — a small in-memory CRUD for users. Demonstrates @RestController, every mapping verb, all four parameter decorators, and the typed-error pattern.

Source

import {
  RestController, GetMapping, PostMapping, PutMapping, DeleteMapping,
  RequestParam, PathVariable, RequestBody, RequestHeader,
  HttpError,
} from 'annotify';

interface User {
  id: number;
  name: string;
}

const users: User[] = [
  { id: 1, name: 'Ada' },
  { id: 2, name: 'Grace' },
];

@RestController('/users')
export class UserController {

  @GetMapping('/')
  list(@RequestParam('limit', '10') limit: string) {
    const n = Math.max(0, Math.min(users.length, Number(limit) || users.length));
    return { count: users.length, items: users.slice(0, n) };
  }

  @GetMapping('/:id')
  get(@PathVariable('id') id: string) {
    const user = users.find((u) => u.id === Number(id));
    if (!user) throw new HttpError(404, `User ${id} not found`);
    return user;
  }

  @PostMapping('/')
  create(
    @RequestBody() body: User,
    @RequestHeader('X-Token') token: string,
  ) {
    if (!token) throw new HttpError(400, 'X-Token header is required');
    if (!body || typeof body.id !== 'number' || typeof body.name !== 'string') {
      throw new HttpError(400, 'Body must be { id: number, name: string }');
    }
    users.push(body);
    return { created: body, echoedToken: token };
  }

  @PutMapping('/:id')
  update(
    @PathVariable('id') id: string,
    @RequestBody() body: User,
  ) {
    const i = users.findIndex((u) => u.id === Number(id));
    if (i < 0) throw new HttpError(404, `User ${id} not found`);
    users[i] = body;
    return users[i];
  }

  @DeleteMapping('/:id')
  remove(@PathVariable('id') id: string) {
    const i = users.findIndex((u) => u.id === Number(id));
    if (i < 0) throw new HttpError(404, `User ${id} not found`);
    const [removed] = users.splice(i, 1);
    return { deleted: removed };
  }
}

Annotation commentary

  • @RestController('/users') — base path /users + flag for JSON serialization.
  • @RequestParam('limit', '10') — second argument is the default value, used when the query parameter is absent or empty.
  • @PathVariable('id') — explicitly named, so the resolver does not need to guess.
  • @RequestBody() takes no arguments — JSON is the only body format.
  • @RequestHeader('X-Token') — header lookup is case-insensitive; passing the canonical form is convention.
  • throw new HttpError(404, …) — surfaces a typed error that the server turns into the right HTTP response.

ProductController

examples/product.controller.ts — demonstrates class-level @Controller (without Rest), an optional query parameter with default, and multiple path variables on the same route.

Source

import { Controller, GetMapping, PathVariable, RequestParam } from 'annotify';

interface Product {
  id: string;
  name: string;
}

const products: Product[] = [
  { id: '1', name: 'Widget' },
  { id: '2', name: 'Gadget' },
];

@Controller('/api/v1/products')
export class ProductController {

  @GetMapping('/')
  all(@RequestParam('q') q: string = '') {
    if (!q) return products;
    return products.filter((p) => p.name.toLowerCase().includes(q.toLowerCase()));
  }

  @GetMapping('/:id')
  one(@PathVariable('id') id: string) {
    return products.find((p) => p.id === id) ?? null;
  }

  @GetMapping('/:id/reviews/:reviewId')
  review(
    @PathVariable('id') id: string,
    @PathVariable('reviewId') reviewId: string,
  ) {
    return { productId: id, reviewId, body: `Review ${reviewId} of product ${id}` };
  }
}

Routes registered

MethodPathHandler
GET/api/v1/productsall(q)
GET/api/v1/products/:idone(id)
GET/api/v1/products/:id/reviews/:reviewIdreview(id, reviewId)

Annotation commentary

  • @Controller('/api/v1/products') — the /api/v1 prefix is part of the class-level path, demonstrating that you can put API versioning in one place.
  • @RequestParam('q') q: string = '' — the TypeScript default is a runtime fallback too: if the query parameter is absent, the resolver passes undefined and the default kicks in.
  • Multiple path variables on a single route are matched in order from the URL.

Idioms

Validate inside the handler. annotify does not impose a validation layer. Validate inputs in the handler body and throw HttpError(400, ...) on bad input. Pulling in Zod or similar is fine — that would be a userland concern.

Return plain JSON. Whatever your handler returns is serialized as JSON (when defaultIsRest is true, which is the default). Return an object; the framework handles JSON.stringify.

Module-scoped state is fine. The controllers here use top-level const users and const products arrays. They survive across requests within the same Node process, so the demo's CRUD operations feel real. In a production app, swap this for a database connection.
