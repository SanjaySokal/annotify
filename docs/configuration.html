Configuration

Reference for tsconfig.json and package.json setup when using annotify.

tsconfig.json

annotify requires legacy TypeScript decorators. Use this as a starting point:

{
  "compilerOptions": {
    "target": "ES2022",
    "lib": ["ES2022"],
    "module": "NodeNext",
    "moduleResolution": "NodeNext",
    "rootDir": "./",
    "outDir": "./dist",
    "experimentalDecorators": true,
    "emitDecoratorMetadata": false,
    "useDefineForClassFields": false,
    "strict": true,
    "esModuleInterop": true,
    "skipLibCheck": true,
    "forceConsistentCasingInFileNames": true,
    "declaration": true,
    "declarationMap": true,
    "sourceMap": true,
    "types": ["node"]
  },
  "include": ["src/**/*"]
}

Option reference

OptionRecommendedWhy
experimentalDecorators true Without this, parameter decorators (@PathVariable, @RequestBody, @RequestParam, @RequestHeader) are stripped from the compiled output. The decorators still type-check, but they have no runtime effect.
emitDecoratorMetadata false annotify does not use Reflect.metadata. Setting this to true would emit type metadata you don't need and increase output size.
useDefineForClassFields false With this true, class fields are defined via Object.defineProperty and decorators on those fields may not apply. false lets class fields initialize in the standard ES way that decorators expect.
target "ES2022" annotify's output uses standard class fields and async iteration. ES2022 covers both.
module / moduleResolution "NodeNext" Matches annotify's "type": "module". The scanner uses pathToFileURL + import(), which requires real ESM.
strict true annotify's own type signatures are designed for strict mode.
esModuleInterop true Lets you import CommonJS modules with default-import semantics if needed.
skipLibCheck true Skips type checking inside node_modules; faster builds.
declaration true If you're publishing your own package that uses annotify, you want .d.ts files emitted alongside the JS.
sourceMap / declarationMap true Helpful for stack traces to point at your TypeScript source.

package.json

For a project that consumes annotify:

{
  "name": "my-app",
  "version": "1.0.0",
  "private": true,
  "type": "module",
  "scripts": {
    "build": "tsc",
    "start": "node dist/main.js",
    "dev": "tsc --watch"
  },
  "dependencies": {
    "annotify": "^0.1.0"
  },
  "devDependencies": {
    "@types/node": "^22.0.0",
    "typescript": "^5.6.0"
  }
}

Notes

  • "type": "module" — required so import / export statements work without file extensions everywhere. annotify is itself ESM.
  • "scripts" — build runs tsc; start runs the compiled output. dev uses tsc --watch for live rebuilding.
  • "dependencies" — annotify only. Zero transitive deps.
  • "devDependencies" — typescript and @types/node.

Environment variables

The default main.ts honors:

VariableDefaultPurpose
PORT3000Port for AppBuilder.listen()
HOST127.0.0.1Host binding

You can replace this with anything — AppBuilder.listen(port, host?) takes both explicitly.
