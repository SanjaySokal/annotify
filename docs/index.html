annotify

Annotation-based HTTP routing for Node.js + TypeScript — Spring Boot feel, zero runtime dependencies.

annotify brings @RestController, @GetMapping, @PathVariable, @RequestBody, and the rest of the Spring routing vocabulary to Node's native http module. No Express, no Koa, no router libraries — just decorators and a small framework.

Annotation-driven

Decorate classes and methods. The router reads metadata at registration time.

Zero deps

No Express, no Koa, no router libs. Only typescript as a dev-dependency.

Auto-discovery

Point at a directory; controllers are imported and bound automatically.

Trie router

Fast matching with literal-segment priority and path-variable fallback.

Typed errors

Throw HttpError from handlers — proper status codes and JSON bodies.

Spring parity

Mapping, PathVariable, RequestParam, RequestBody, RequestHeader — the essentials.

Quick example

import {
  RestController, GetMapping, PostMapping,
  PathVariable, RequestBody, RequestHeader, HttpError,
} from 'annotify';

@RestController('/users')
export class UserController {

  @GetMapping('/:id')
  get(@PathVariable('id') id: string) {
    return { id, name: 'Ada' };
  }

  @PostMapping('/')
  create(
    @RequestBody() body: { name: string },
    @RequestHeader('X-Token') token: string,
  ) {
    if (!token) throw new HttpError(401, 'X-Token required');
    return { created: body, token };
  }
}

Bind it in main.ts:

import { AppBuilder } from 'annotify';
const app = new AppBuilder();
await app.scan('./dist/controllers');
await app.listen(3000);

What's new in 0.5.0

  • AppBuilder.useInterceptor(fn) — register a function that runs against every controller class before instantiation. The new cross-cutting hook that lets external packages (caching, transactions, auth) mutate the controller prototype to wire in behavior. See Interceptors.
  • Param-resolution fix — resolveArgs now reads entry.paramTypes.length (the canonical metadata) instead of fn.length (which collapses to 0 under Function.prototype.bind and any wrapper that uses ...args). This unblocks interceptor / decorator wrappers — previously, a wrapper around a handler would silently pass undefined for every parameter.

What's new in 0.3.0

  • Request logger — app.useLogger({ enabled: true }) emits one access-log line per request (method, path, status, duration). See Request logger.
  • Routes introspection — app.exposeRoutes() exposes GET /__annotify/routes returning the full route table as JSON. See Routes introspection.
  • Smoke test count: 15 — two new tests cover logger output and the introspection endpoint.

Why legacy TypeScript decorators

annotify uses experimentalDecorators: true rather than TC39 Stage 3. The reason is pragmatic: TC39 Stage 3 decorators do not include parameter decorators in the current proposal, and TypeScript 5 strips them from the compiled output when experimentalDecorators is off. Without parameter decorators there is no way to express @PathVariable, @RequestBody, etc. — they would simply not exist at runtime.

When TC39 ships parameter decorators and TypeScript implements them, annotify can migrate to Stage 3 without changing its public API.
