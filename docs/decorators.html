Decorators

Every decorator exported from annotify, with signatures, behavior, and examples.

Import surface

import {
  // class decorators
  Controller, RestController,

  // method decorators
  RequestMapping, GetMapping, PostMapping,
  PutMapping, DeleteMapping, PatchMapping,

  // parameter decorators
  RequestParam, PathVariable, RequestBody, RequestHeader,

  // runtime
  AppBuilder, HttpError, Router, scanControllers,
} from 'annotify';

Class decorators

@Controller(path?)#

Marks a class as a controller. The optional path is prepended to every route declared on its methods.

@Controller('/api/v1/products')
export class ProductController {
  @GetMapping('/')          // → GET /api/v1/products
  all() { return products; }

  @GetMapping('/:id')       // → GET /api/v1/products/:id
  one(@PathVariable('id') id: string) { return products[id]; }
}

@RestController(path?)#

Same as @Controller, but additionally sets the isRest flag. With AppBuilder.setDefaultIsRest(true) (the default), all controllers are treated as JSON endpoints anyway, but the flag is exposed for future per-controller serialization overrides.

Method decorators (route mapping)

Each maps a method to an HTTP verb + path. The path is joined with the class-level prefix.

DecoratorHTTP methodExample
@GetMapping(path?)GET@GetMapping('/users')
@PostMapping(path?)POST@PostMapping('/users')
@PutMapping(path?)PUT@PutMapping('/:id')
@DeleteMapping(path?)DELETE@DeleteMapping('/:id')
@PatchMapping(path?)PATCH@PatchMapping('/:id')
@RequestMapping(path, method)any@RequestMapping('/foo', 'OPTIONS')

All path arguments are optional — @GetMapping() matches the class prefix exactly. Trailing slashes are normalized away: @GetMapping('/users/') on a @Controller('/api') class becomes GET /api/users.

Path variables use a colon prefix: /users/:id. They are URL-decoded automatically.

@RestController('/orders')
export class OrderController {

  @GetMapping('/:orderId/items/:itemId')
  item(
    @PathVariable('orderId') orderId: string,
    @PathVariable('itemId')  itemId: string,
  ) {
    return { orderId, itemId };
  }
}

Parameter decorators

Parameter decorators bind parts of the incoming HTTP request to handler arguments. Their order in the method signature does not need to match the route declaration order — the resolver uses positional metadata captured at decoration time.

@PathVariable(name?)#

Reads a path variable from the matched route. Pass the variable name (without the colon). If you omit the name, the resolver falls back to the first :var it finds in the pattern — explicit names are recommended.

@GetMapping('/users/:id')
get(@PathVariable('id') id: string) { /* ... */ }

@RequestParam(name?, defaultValue?)#

Reads a URL query parameter. The name is the key in the query string. If absent or empty, the optional defaultValue is used. Repeated query keys collapse to the first value.

The decorator has two overloads:

@RequestParam('limit', '10')                                       // positional
@RequestParam({ name: 'limit', defaultValue: '10' })           // options object

@RequestBody()#

Parses the request body as JSON and binds it to the parameter. If Content-Type is not application/json, the server responds with 415 Unsupported Media Type before the handler runs. Bodies larger than 1 MB trigger 413 Payload Too Large.

@PostMapping('/users')
create(@RequestBody() body: User) { /* ... */ }

@RequestHeader(name?)#

Reads a single request header. Header lookup is case-insensitive (HTTP headers are normalized to lowercase by Node).

@PostMapping('/login')
login(@RequestHeader('Authorization') auth: string) { /* ... */ }

@ResponseStatus(code)#

Method decorator. Sets the HTTP status code returned by the handler. Default is 200 for handlers that return a value, or 204 for handlers that return undefined.

@PostMapping('/')
@ResponseStatus(201)
create(@RequestBody() body: User) {
  // any non-throw return → status will be 201
  return body;
}

@CrossOrigin(opts?)#

Method or class decorator. Adds CORS headers to responses and auto-handles OPTIONS preflight. See CORS below for full details.

@CrossOrigin({
  origins: ['http://localhost:5173'],
  credentials: true,
  maxAge: 3600,
})
@RestController('/users')
export class UserController { /* ... */ }

Positional fallback

If a parameter has no decorator at all, the resolver applies a positional fallback:

IndexValue
0IncomingMessage (the raw req)
1ServerResponse (the raw res)
2RequestContext (the full context)

This is useful for low-level handlers that need direct access to the underlying Node objects:

@GetMapping('/stream')
stream(req: IncomingMessage, res: ServerResponse) {
  res.setHeader('Content-Type', 'text/event-stream');
  res.write('data: hello\n\n');
  // Note: returning a value won't be serialized — you've already written.
}

Throwing errors

Handlers can throw HttpError from annotify. The server converts these to JSON responses with the given status code.

import { HttpError } from 'annotify';

@GetMapping('/:id')
get(@PathVariable('id') id: string) {
  const user = users.find(u => u.id === Number(id));
  if (!user) throw new HttpError(404, `User ${id} not found`);
  return user;
}

Non-HttpError exceptions become 500 Internal Server Error.

CORS

@CrossOrigin attaches CORS headers to responses and auto-handles OPTIONS preflight. It works at class level (default for every route in the controller) or method level (override for one route).

@CrossOrigin({
  origins: ['http://localhost:5173'],
  credentials: true,
  maxAge: 3600,
})
@RestController('/users')
export class UserController { /* ... */ }

Shorthand forms:

@CrossOrigin()                                  // '*' for all origins
@CrossOrigin('https://app.example.com')        // single origin
@CrossOrigin(['https://a.com', 'https://b.com'])
@CrossOrigin({ origins: '*', methods: ['GET', 'POST'] })

What it does

  • Adds Access-Control-Allow-Origin (and Vary: Origin when origins is a list).
  • Adds Access-Control-Allow-Methods (defaults to all methods registered at the path).
  • Adds Access-Control-Allow-Headers (configurable).
  • Adds Access-Control-Allow-Credentials: true if credentials: true.
  • Adds Access-Control-Max-Age if maxAge is set.
  • Adds Access-Control-Expose-Headers if exposedHeaders is set.
  • Auto-replies to OPTIONS preflight at known paths with 204 + all of the above.

Order of decorator execution

Important. TypeScript legacy decorators run method decorators before class decorators. This means a method decorator cannot read the class-level basePath at decoration time. annotify handles this by storing the method-level sub-path on each route entry and resolving the full path (basePath + subPath) at registration time inside Router.addController. If you author a custom decorator, follow the same pattern — store raw inputs, resolve later.
