Decorators

src/decorators/metadata.ts

The metadata layer. Defines the WeakMap side-channel used by parameter decorators, and the helpers ensureRouteMeta, getRouteMeta, getOrCreateParamRegistry, consumeParamRegistry.

export const ROUTE_METADATA = Symbol.for('annotify.route');

const paramRegistry = new WeakMap<object, Record<string, ParamMeta[]>>();

export function ensureRouteMeta(ctor: object): RouteMetadata {
  if (!(ctor as any)[ROUTE_METADATA]) {
    Object.defineProperty(ctor, ROUTE_METADATA, {
      value: { basePath: '', isRest: false, routes: [] },
      writable: true,
      configurable: true,
      enumerable: false,
    });
  }
  return (ctor as any)[ROUTE_METADATA];
}

export function consumeParamRegistry(proto: object, methodName: string): ParamMeta[] {
  const reg = paramRegistry.get(proto);
  const params = reg?.[methodName];
  if (reg) delete reg[methodName];
  return params ?? [];
}

src/decorators/path.ts

Two pure functions — joinPath and splitSegments — plus a private normalize. They share responsibility for the path semantics (trailing-slash handling, leading-slash enforcing) that all routing decorators rely on.

src/decorators/types.ts

Local ClassDecorator, MethodDecorator, ParameterDecorator types matching the legacy TS signature. We define them locally so the project doesn't depend on the (slightly off) types in lib.es5.d.ts.

src/decorators/controller.ts

Just two factory functions: @Controller(path?) and @RestController(path?). Both call ensureRouteMeta(ctor) and set basePath; the latter additionally sets isRest = true.

src/decorators/params.ts

Four parameter decorator factories (@RequestParam, @PathVariable, @RequestBody, @RequestHeader). Each returns a function that, when invoked with the (target, propertyKey, parameterIndex) tuple, writes a ParamMeta to the registry.

export function PathVariable(name?: string): ParameterDecorator {
  return (target, propertyKey, parameterIndex) => {
    const reg = getOrCreateParamRegistry(target);
    const list = (reg[String(propertyKey)] ??= []);
    list[parameterIndex] = { kind: 'path', name };
  };
}

src/decorators/mapping.ts

The five HTTP-verb decorators + the generic @RequestMapping. Each makeMapping(method, path) returns a method decorator that snapshots the per-parameter metadata accumulated in the side-channel and pushes a RouteEntry onto meta.routes. The full path is resolved later by Router.addController.

src/decorators/index.ts

Internal barrel re-export for the decorator modules.

Core

src/core/route-node.ts

Plain data class for trie nodes. One segment string, a map of literal children, an optional varChild, and a map of HTTP-verb handlers.

src/core/router.ts

The trie router. addEntry walks down the path creating nodes as needed and registers the handler at the final node. match(method, path) walks down the request path trying literals first, falling back to varChild. methodsAt(path) is for 405 detection.

addController(meta, instance) {
  for (const entry of meta.routes) {
    entry._handler = (instance as any)[entry.handlerName].bind(instance);
    entry.path = joinPath(meta.basePath, entry.subPath);  // deferred join
    this.addEntry(entry);
  }
}

src/core/resolver.ts

Reads entry.paramTypes[i] for each parameter position on the bound handler, dispatches by kind, and assembles the args array.

src/core/context.ts

Defines RequestContext — the bag passed to resolvers:

export interface RequestContext {

  pathVars: Record<string, string>;

src/core/errors.ts

The HttpError class plus a handful of private sendX helpers.

export class HttpError extends Error {
  status: number;
  details?: unknown;
  constructor(status: number, message: string, details?: unknown) {
    super(message);
    this.status = status;
    this.details = details;
  }
}

src/core/scanner.ts

Walks a directory, filters .js / .mjs, converts paths to file URLs, dynamically imports each module, and collects any export whose constructor has ROUTE_METADATA.

src/core/server.ts

The createApp(router, opts) factory. Reads the request body, walks the router, resolves arguments, calls the bound handler, and funnels errors through sendInternalError.

src/core/app.ts

The AppBuilder façade. Holds a list of controllers and a router, builds on demand, and exposes listen(port, host?).

App entry

src/main.ts

import { AppBuilder } from './index.js';
import { fileURLToPath } from 'node:url';
import { dirname, resolve } from 'node:path';

const __dirname = dirname(fileURLToPath(import.meta.url));

const PORT = Number(process.env.PORT ?? 3000);
const HOST = process.env.HOST ?? '127.0.0.1';

const app = new AppBuilder();
const controllersDir = resolve(__dirname, '..', 'examples');

await app.scan(controllersDir);
const server = await app.listen(PORT, HOST);

const shutdown = (signal: string) => {
  console.log(`\n[server] ${signal} received, shutting down...`);
  server.close(() => process.exit(0));
  setTimeout(() => process.exit(1), 5000).unref();
};
process.on('SIGINT', () => shutdown('SIGINT'));
process.on('SIGTERM', () => shutdown('SIGTERM'));
