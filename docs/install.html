Install

Three commands and you're up. annotify is a single npm dependency with zero transitive deps.

Requirements

  • Node.js 18 or newer (developed and tested on 22 / 26)
  • npm, pnpm, or yarn — examples below use npm
  • TypeScript 5.0+ as a dev-dependency

Install annotify

npm install annotify

That's the runtime dependency. annotify ships ES modules with TypeScript declarations.

Install dev dependencies

npm install --save-dev typescript @types/node

TypeScript compiles your controllers; @types/node gives you types for http, fs, etc.

Zero runtime deps. annotify pulls in nothing — no Express, no Koa, no router libs. The only transitive cost in your node_modules is the package itself plus what your code itself imports.

tsconfig.json

annotify requires legacy TypeScript decorators. Start with this:

{
  "compilerOptions": {
    "target": "ES2022",
    "lib": ["ES2022"],
    "module": "NodeNext",
    "moduleResolution": "NodeNext",
    "rootDir": "./",
    "outDir": "./dist",
    "experimentalDecorators": true,
    "emitDecoratorMetadata": false,
    "useDefineForClassFields": false,
    "strict": true,
    "esModuleInterop": true,
    "skipLibCheck": true,
    "forceConsistentCasingInFileNames": true,
    "declaration": true,
    "declarationMap": true,
    "sourceMap": true,
    "types": ["node"]
  },
  "include": ["src/**/*"]
}

The pivotal options:

OptionValueWhy
experimentalDecoratorstrueWithout this, parameter decorators (@PathVariable, etc.) are stripped from the compiled output.
emitDecoratorMetadatafalseannotify has its own side-channel — no reflect-metadata needed.
useDefineForClassFieldsfalseEnsures decorator-applied class fields behave correctly.
module / moduleResolutionNodeNextMatches annotify's "type": "module".
declarationtrueEmits .d.ts files for your own consumers.

See Configuration for full details on each option.

package.json scripts

{
  "name": "my-app",
  "version": "1.0.0",
  "private": true,
  "type": "module",
  "scripts": {
    "build": "tsc",
    "start": "node dist/main.js"
  },
  "dependencies": {
    "annotify": "^0.1.0"
  },
  "devDependencies": {
    "@types/node": "^22.0.0",
    "typescript": "^5.6.0"
  }
}

Verify the install

Once installed, this should run without errors:

import { AppBuilder, RestController, RequestLogger } from 'annotify';
console.log(typeof RestController);  // 'function'
console.log(typeof AppBuilder);      // 'function'
console.log(typeof RequestLogger);    // 'function'

Then head to Quick start to write your first controller. Once it's running, turn on the logger with app.useLogger({ enabled: true }) and expose your routes with app.exposeRoutes().
